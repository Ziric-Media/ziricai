# API
> Category: api | ZiricAI Platform Knowledge Base

## Q: Does ZiricAI have an API?
**A:** Yes — REST API for contacts, conversations, knowledge, and webhook subscriptions.

## Q: How do I get API keys?
**A:** Settings → Developer → generate keys with scoped permissions.

## Q: What are API rate limits?
**A:** Trial 1,000 calls/month; Starter 5,000; Professional 25,000; Business 50,000; Enterprise unlimited.

## Q: Is API documentation available?
**A:** Developer docs in portal cover auth, endpoints, and examples.

## Q: Can I create contacts via API?
**A:** POST contacts into CRM from web forms or external systems.

## Q: Can I read conversation history?
**A:** GET transcripts for BI tools with proper authentication.

## Q: Are webhooks part of API?
**A:** Subscribe to message.received and crm.updated events.

## Q: Is OAuth supported?
**A:** API keys for server-to-server; OAuth for partner apps on Enterprise.

## Q: Is the API RESTful?
**A:** JSON REST over HTTPS with standard HTTP status codes.

## Q: Can I upload knowledge via API?
**A:** Programmatic document upload for CMS-synced knowledge bases.

## Q: Are sandbox keys available?
**A:** Trial tenants use same API with lower limits for testing.

## Q: How is API secured?
**A:** TLS 1.3, API key rotation, and IP allowlists on Enterprise.

## Q: Can agencies use multi-tenant API?
**A:** Enterprise partners manage multiple tenant IDs via scoped credentials.

## Q: What happens if I exceed API limits?
**A:** 429 responses and portal warnings; upgrade for higher caps.

## Q: Is there a Postman collection?
**A:** Download collection from Documentation section in portal.

## Q: Can API trigger automations?
**A:** External events via webhook inbound endpoints fire workflows.

## Q: Are API changes versioned?
**A:** Version prefix in URL; deprecations announced in Product Updates.

## Q: Can I build a custom portal?
**A:** API plus webhooks enable custom dashboards — Enterprise support available.

## Q: Is personal data exposed via API?
**A:** POPIA/GDPR rules apply — access only authorized fields with consent.

## Q: Where are API logs?
**A:** Developer console shows recent requests and errors for debugging.
